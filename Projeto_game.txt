1. Visão Geral do Projeto

“Fragmentos do Tempo” é um jogo de plataforma 2D focado em manipulação temporal.
O diferencial central é a possibilidade de voltar alguns segundos no tempo e criar uma versão passada do personagem que repete suas ações.

A proposta é usar essa mecânica não apenas como efeito visual, mas como base para puzzles, desafios técnicos e exploração de sistemas complexos de estado.

O projeto será desenvolvido ao longo de 1 ano, começando com uma base simples e evoluindo gradualmente para um sistema mais robusto e organizado.

2. Objetivos do Projeto
Objetivo Principal

Desenvolver um jogo jogável e tecnicamente sólido que explore manipulação de tempo com foco em organização de código, estrutura modular e controle de estados.

Objetivos Secundários

Aprimorar conhecimento em física 2D

Trabalhar com armazenamento e recuperação de estados

Aprender a organizar projeto de médio prazo

Aplicar boas práticas de versionamento (Git)

Desenvolver documentação técnica

3. Conceito do Jogo

O jogador controla um personagem preso em um ambiente fragmentado no tempo.

Ele pode:

Voltar até 3 segundos no tempo

Criar um “eco temporal” (clone) que repete suas ações passadas

O jogador precisa coordenar suas próprias versões para resolver obstáculos como:

Plataformas móveis

Portas com pressão simultânea

Sincronização de mecanismos

A ideia é que o jogo ensine o jogador a “pensar em camadas temporais”.

4. Estrutura Técnica Principal

O projeto será organizado em sistemas independentes:

4.1 Sistema de Movimento

Andar

Pular

Controle de aceleração

Colisão com física estável

4.2 Sistema de Registro Temporal

Armazenamento contínuo de:

Posição

Velocidade

Estado de animação

Inputs

Uso de buffer circular para limitar memória

4.3 Sistema de Rewind

Leitura reversa dos estados gravados

Desativação temporária de colisões instáveis

Garantia de consistência física

4.4 Sistema de Clone Temporal

Reprodução exata dos inputs gravados

Independência do jogador atual

Interação física com o cenário

5. Planejamento Anual (Dividido por Fases)
FASE 1 — Fundamentos (Meses 1–3)

Objetivo: Criar base sólida.

Movimento 2D refinado

Sistema de câmera

Estrutura de projeto organizada

Primeira fase simples

Documentação técnica inicial

Foco principal: qualidade da base antes de adicionar complexidade.

FASE 2 — Manipulação Temporal Básica (Meses 4–6)

Objetivo: Implementar rewind funcional.

Sistema de gravação de estados

Rewind simples (sem clone ainda)

Testes de estabilidade

Ajustes na física

Aqui começa o maior desafio técnico: manter consistência sem bugs graves.

FASE 3 — Clone Temporal (Meses 7–9)

Objetivo: Implementar eco temporal jogável.

Reprodução de inputs

Sistema de sincronização

Correção de colisões múltiplas

Primeiros puzzles baseados em clone

Essa fase exige organização e testes constantes.

FASE 4 — Polimento e Expansão (Meses 10–12)

Objetivo: Transformar protótipo em jogo completo.

6 a 10 fases progressivas

Introdução de novos elementos:

Plataformas que existem apenas em uma linha temporal

Interruptores que exigem múltiplos clones

Interface básica

Sistema de menu

Sistema de checkpoint

Otimização

Entrega final: versão jogável estável.

6. Escopo Final Esperado

8 a 12 fases

Sistema de rewind estável

Sistema de clone funcional

Design progressivo de dificuldade

Código modular organizado

Documento técnico complementar

7. Principais Desafios Técnicos

Controle de memória no armazenamento de estados

Evitar inconsistências físicas durante rewind

Garantir que clones não quebrem colisões

Manter performance estável

8. Estratégias para Evitar Problemas

Testar cada sistema isoladamente

Manter documentação atualizada

Usar controle de versão desde o início

Criar builds periódicas

Não aumentar escopo antes da base estar sólida

9. Aprendizados Esperados

Ao final do projeto, espera-se domínio em:

Estruturação de projetos de médio prazo

Controle de estados complexos

Organização modular

Lógica temporal aplicada

Planejamento de desenvolvimento

10. Considerações Finais

“Fragmentos do Tempo” não é apenas um jogo de plataforma, mas um estudo prático sobre controle de estado, memória e organização de sistemas interdependentes.

O projeto foi planejado para crescer de forma sustentável ao longo de 1 ano, priorizando estabilidade e clareza estrutural antes de complexidade excessiva.

O foco principal não é apenas terminar um jogo, mas evoluir como desenvolvedor durante o processo.
