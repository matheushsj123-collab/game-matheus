5\. Planejamento Anual (Dividido por Fases)

FASE 1 — Fundamentos (Meses 1–3)



Objetivo: Criar base sólida.



Movimento 2D refinado



Sistema de câmera



Estrutura de projeto organizada



Primeira fase simples



Documentação técnica inicial



Foco principal: qualidade da base antes de adicionar complexidade.



FASE 2 — Manipulação Temporal Básica (Meses 4–6)



Objetivo: Implementar rewind funcional.



Sistema de gravação de estados



Rewind simples (sem clone ainda)



Testes de estabilidade



Ajustes na física



Aqui começa o maior desafio técnico: manter consistência sem bugs graves.



FASE 3 — Clone Temporal (Meses 7–9)



Objetivo: Implementar eco temporal jogável.



Reprodução de inputs



Sistema de sincronização



Correção de colisões múltiplas



Primeiros puzzles baseados em clone



Essa fase exige organização e testes constantes.



FASE 4 — Polimento e Expansão (Meses 10–12)



Objetivo: Transformar protótipo em jogo completo.



6 a 10 fases progressivas



Introdução de novos elementos:



Plataformas que existem apenas em uma linha temporal



Interruptores que exigem múltiplos clones



Interface básica



Sistema de menu



Sistema de checkpoint



Otimização



Entrega final: versão jogável estável.



6\. Escopo Final Esperado



8 a 12 fases



Sistema de rewind estável



Sistema de clone funcional



Design progressivo de dificuldade



Código modular organizado



Documento técnico complementar



7\. Principais Desafios Técnicos



Controle de memória no armazenamento de estados



Evitar inconsistências físicas durante rewind



Garantir que clones não quebrem colisões



Manter performance estável



8\. Estratégias para Evitar Problemas



Testar cada sistema isoladamente



Manter documentação atualizada



Usar controle de versão desde o início



Criar builds periódicas



Não aumentar escopo antes da base estar sólida

Planejamento de desenvolvimento



10\. Considerações Finais



“Fragmentos do Tempo” não é apenas um jogo de plataforma, mas um estudo prático sobre controle de estado, memória e organização de sistemas interdependentes.



O projeto foi planejado para crescer de forma sustentável ao longo de 1 ano, priorizando estabilidade e clareza estrutural antes de complexidade excessiva.



O foco principal não é apenas terminar um jogo, mas evoluir como desenvolvedor durante o processo.
